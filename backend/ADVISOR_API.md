# Advisor Insights API

Financial advisory endpoint providing business insights and metrics for authenticated users.

## Endpoint

```
GET /api/advisor/insights?period=this_month
```

### Query Parameters

- `period` (optional, default: `this_month`)
  - `this_month` - Current calendar month
  - `last_month` - Previous calendar month
  - `last_30_days` - Last 30 days from today
  - `this_week` - Current calendar week (Monday-Sunday)

### Metrics Explained

- **totalSales**: Sum of all unpaid + paid SELL invoices in period
- **totalExpenses**: Sum of all unpaid + paid BUY invoices in period
- **netProfit**: `totalSales - totalExpenses`
- **receivablesTotal**: Unpaid SELL invoices (money customers owe)
- **payablesTotal**: Unpaid BUY invoices (money owed to suppliers)
- **overdueReceivablesTotal**: Receivables past due date
- **overduePayablesTotal**: Payables past due date
- **unpaidSalesInvoicesCount**: Number of pending/unpaid SELL invoices
- **overdueSalesInvoicesCount**: Number of overdue SELL invoices

### Insight Types

- `critical` - Urgent action required (e.g., overdue receivables, negative profit)
- `warning` - Important alert (e.g., high expenses, large single debt)
- `opportunity` - Positive opportunity (e.g., top-selling product)
- `info` - Informational (e.g., trend notes)

### Source

- `ai` - Insights generated by AI model (when API key configured and available)
- `rules` - Insights generated by rule-based engine (fallback when AI unavailable)

## Configuration

### AI Provider

The endpoint uses OpenRouter by default (via `OPENROUTER_API_KEY` env var).

**Fallback behavior:**
- If no API key configured → uses rule-based insights only
- If AI API fails (rate limit, timeout) → falls back to rule-based insights
- Both modes return the same response format

## Privacy

**Data sent to AI model:**
- Aggregated financial metrics only
- Anonymized client references (e.g., `client_1`, `client_2`)
- Anonymized product references (e.g., `item_1`, `item_2`)

**Never sent to AI:**
- Real client names
- Phone numbers, emails, addresses
- Invoice notes or raw text
- Customer bank details

**Mapping:** Backend maintains local anonymization mapping; AI sees only refs, not real identities.
